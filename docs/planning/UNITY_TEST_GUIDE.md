## 📋 测试前准备

### 环境要求

| 项目 | 说明 |
|------|------|
| Unity 版本 | 与项目一致（查看 `ProjectSettings/ProjectVersion.txt`）|
| 项目路径 | `/root/game_sxcg/` |
| 场景文件 | `Assets/Scenes/MainScene.unity`（唯一主场景）|

### 打开项目的步骤

1. 打开 Unity Hub → 点击「添加」→ 选择 `game_sxcg` 文件夹
2. 在 Unity Hub 中双击项目打开
3. 等待编辑器加载完毕（右下角进度条消失）
4. 在 **Hierarchy** 窗口中确认 `MainScene` 已加载
5. 如果 Hierarchy 为空 → 菜单栏 `File` → `Open Scene` → 选择 `Assets/Scenes/MainScene.unity`

### 如何查看 Console 日志

> **这是测试过程中最重要的窗口！** 所有验证结论都通过 Console 日志确认。

1. 菜单栏 `Window` → `General` → `Console`（或快捷键 `Ctrl+Shift+C`）
2. 点击 Console 窗口左上角的 **Clear** 按钮，清除旧日志
3. 测试过程中，重点关注以下关键日志：
   - `[肉鸽]` 开头 → 肉鸽系统运行状态
   - `[StateMachine]` 开头 → 状态转换流程
   - `[SaveSystem]` 开头 → 存档/读档操作
   - `[CardMergeSystem]` 开头 → 卡牌合成
   - `[随机事件]` 开头 → 事件系统
   - `[LeaderboardManager]` 开头 → 排行榜
   - `🔴 红色图标` → 错误（Error），代表有问题
   - `🟡 黄色图标` → 警告（Warning），可关注但不一定有问题

## 测试一：完整游戏闭环（选英雄 → 掷骰 → 战斗 → 结算 → 肉鸽循环）

> 验证目标：从主菜单开始，完成一局完整的 肉鸽（Roguelike）循环。

### 步骤 1-1：启动游戏，进入主菜单

| 操作 | 说明 |
|------|------|
| **点击** | Unity Editor 顶部中央的 **▶（播放）按钮** |
| **看什么** | Game 窗口显示主菜单界面 |
| **预期结果** | Console 输出 `[StateMachine] 进入主菜单` |

### 步骤 1-2：开始新游戏

| 操作 | 说明 |
|------|------|
| **点击** | 主菜单上的 **「开始游戏」按钮** |
| **看什么** | 界面切换到英雄选择界面 |

### 步骤 1-3：选择英雄

| 操作 | 说明 |
|------|------|
| **点击** | 在英雄选择界面中，点击 **战士 / 法师 / 刺客** 其中一个英雄卡片 |

### 步骤 1-4：地图节点选择

| 操作 | 说明 |
|------|------|
| **点击** | 在地图界面中，点击一个 **高亮的可点击节点**（通常是第一层） |

## 测试二：肉鸽存档恢复（BE-22 新增）

> 验证目标：中途退出游戏后，能通过存档恢复到之前的肉鸽进度。

### 步骤 2-1：创建一个有进度的存档

1. 按照 **测试一** 的步骤，正常游戏到 **第 3 关以上**
2. 在任意关卡的战斗结算后（选择奖励之后），观察 Console

| 检查点 | 预期结果 |
|--------|----------|
| 自动存档日志 | Console 输出 `[SaveSystem] 存档完成 LvX 金币XXX` |
| 肉鸽运行存档 | Console 输出 `[SaveSystem] 肉鸽运行存档完成 FloorX GoldXXX` |

### 步骤 2-2：模拟中途退出

1. **直接点击 Unity Editor 的 ▶ 按钮停止运行**（模拟中途关闭游戏）
2. Game 窗口画面停止

### 步骤 2-3：重新启动并恢复

1. **再次点击 ▶ 按钮启动游戏**
2. 观察主菜单界面

| 检查点 | 预期结果 |
|--------|----------|
| 「继续游戏」按钮 | 主菜单上出现 **「继续游戏」按钮**（之前没有） |
| 存档检测 | Console 无红色错误 |

3. **点击「继续游戏」按钮**

| 检查点 | 预期结果 |
|--------|----------|
| 恢复日志 | Console 输出 `[SaveSystem] 读档完成 LvX` |
| 肉鸽恢复 | Console 输出 `[肉鸽] 运行恢复完成! Floor=X, Heroes=X, Relics=X, Gold=X, Seed=XXX` |
| 状态恢复 | Console 输出 `[SaveSystem] 状态恢复完成` |
| 界面跳转 | 进入地图选择界面，关卡编号与退出前一致 |

### 步骤 2-4：验证恢复内容准确性

恢复后，逐一核对以下内容：

| 核对项 | 检查方法 | 预期结果 |

| 关卡编号 | 看 Console 日志或界面标题 | 与退出前一致 |
| 英雄列表 | 看队伍栏 | 英雄名称和等级与退出前相同 |
| 金币数量 | 看界面右上角金币数 | 与退出前相同（±0） |
| 地图节点 | 看地图界面 | 已访问节点变灰，可选节点高亮 |

### 步骤 2-5：继续游戏验证

1. 恢复后，继续选择地图节点打一关
2. 正常完成战斗和结算

| 检查点 | 预期结果 |
|--------|----------|
| 能正常继续 | 游戏循环无异常，关卡正常递增 |
| 无报错 | Console 无红色 Error |

### 步骤 2-6：验证存档清除

1. 在游戏中阵亡（GameOver）或通关
2. 回到主菜单后，观察 Console

| 检查点 | 预期结果 |
|--------|----------|

## 测试三：卡牌合成系统（BE-20 新增）

> 验证目标：两张同稀有度卡牌可以合成，获得更高稀有度的随机卡牌。

### 步骤 3-1：获取足够的卡牌和金币

1. 按照 **测试一** 正常游戏，打到第 3-5 关
2. 在奖励选择中，尽量选择 **「新卡牌」** 类型的奖励
3. 积累金币到 **100 金币以上**（合成需要 50 金币/次）

### 步骤 3-2：进入卡牌合成界面

1. 在结算后或地图选择界面，找到并点击 **「卡组」或「卡牌编辑」按钮**
2. 在卡组编辑面板中，找到 **「合成」或「卡牌合成」入口按钮**
3. 点击进入合成界面

| 检查点 | 预期结果 |
|--------|----------|
| 合成面板打开 | 显示卡牌列表和合成槽位 |
| 卡牌稀有度色标 | 卡牌边框/名字显示颜色区分：灰色(White)、蓝色(Blue)、紫色(Purple)、金色(Gold) |

### 步骤 3-3：检查合成前置条件

卡牌合成规则：
- **两张相同稀有度**的卡牌才能合成
- 稀有度晋升链：**White → Blue → Purple → Gold**
- **Gold（金色）** 是最高稀有度，不可再合成
- 需要消耗 **50 金币**

| 检查点 | 预期结果 |
|--------|----------|
| 不同稀有度卡牌 | 合成按钮灰色不可点 |
| 只有一张某稀有度卡 | 合成按钮灰色不可点 |
| 金币不足时 | 合成按钮灰色不可点 |

### 步骤 3-4：执行合成

1. 在卡牌列表中，选择 **两张同稀有度** 的卡牌（如两张 White 卡）
2. 将两张卡牌拖入合成槽（或点击选择）
3. 观察合成预览区域显示可能的结果

| 检查点 | 预期结果 |
|--------|----------|
| 合成预览 | 右侧显示下一稀有度（如 Blue）的随机卡牌池 |
| 合成按钮 | 变为可点击状态（不再灰色） |

4. **点击「合成」按钮**

| 检查点 | 预期结果 |
|--------|----------|
| 合成动画 | 两张卡牌飞入中央 → 融合 → 光效爆发 → 新卡弹出 |
| Console 日志 | 输出 `[CardMergeSystem] 合成成功！XXX(White) + XXX(White) → YYY(Blue)，花费 50 金币` |
| 金币变化 | 金币减少 50 |
| 卡牌变化 | 背包中两张素材卡消失，新增一张更高稀有度的卡牌 |

### 步骤 3-5：验证合成限制

1. 尝试将两张 **Gold（金色）** 卡牌合成

| 检查点 | 预期结果 |
|--------|----------|
| 不可合成 | 合成按钮灰色，Console 输出 `[CardMergeSystem] 无法合成：条件不满足` 或按钮不可点击 |

2. 尝试将 **不同稀有度** 的卡牌合成（如一张 White + 一张 Blue）

| 检查点 | 预期结果 |
|--------|----------|
| 不可合成 | 合成按钮灰色不可点击 |

### ✅ 测试三通过标准

- [ ] 卡牌显示稀有度颜色区分
- [ ] 两张同稀有度卡牌可以合成
- [ ] 合成后获得更高稀有度的卡牌
- [ ] 合成消耗 50 金币
- [ ] 合成后素材卡消失，新卡出现
- [ ] Gold 卡牌无法合成
- [ ] 不同稀有度卡牌无法合成

## 测试五：排行榜（BE-23 新增）

> 验证目标：游戏结束后能提交分数到排行榜，排行榜能正确显示。

### 步骤 5-1：完成一局游戏

1. 按照 **测试一** 正常游戏一局
2. 打到 GameOver（阵亡或通关）

### 步骤 5-2：观察分数提交

在 GameOver 面板出现时，查看 Console 日志：

| 检查点 | 预期结果 |
|--------|----------|
