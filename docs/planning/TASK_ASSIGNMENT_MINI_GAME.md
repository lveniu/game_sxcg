### 工期风险提示

| 风险项 | 说明 | 应对 |
|--------|------|------|
| 总工时85天 vs 可用80天 | 超出5人天（~6%） | P2/P3任务共8人天可延后，实际P0+P1仅74.5天 |
| 机制怪系统6人天 | 复杂度最高、最容易延期 | MVP先做3种（护盾互换/伤害反弹/狂暴），其余后续迭代 |
| 包体4MB限制 | 技术不确定性最高 | W1提前PoC验证，如不达标尽早启动B计划 |
| 前端W5较闲 | 后端集中做机制怪时前端负载低 | 前端可提前推进W6的WebGL构建准备工作 |
