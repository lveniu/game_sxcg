## FE-21 卡牌稀有度UI + 合成界面 [P1] [预估 ~250行]

### 背景
卡牌系统完成度78%，缺稀有度体系和合成升级。卡牌收集是核心驱动力，前端需补UI层。

### 任务
1. **稀有度色标系统**
   - 在 CardDeckEditorPanel 和 CardPlayPanel 的卡牌渲染中添加稀有度边框色
   - 4级稀有度: Common=#808080 / Rare=#4FC3F7 / Epic=#AB47BC / Legendary=#FFD740
   - 卡牌名字颜色同步变化

2. **卡牌合成界面 CardMergePanel**
   - 新面板，从 CardDeckEditorPanel 入口进入
   - 左侧: 卡牌列表（可筛选同稀有度）
   - 中央: 合成槽（拖入2张同稀有度卡牌）
   - 右侧: 合成预览（随机1张更高稀有度）
   - 合成动画: 2张卡牌飞入中央 → 旋转融合 → 光效爆发 → 新卡弹出
   - 按钮状态: 不足2张时灰色不可点

3. **接口依赖**
   - CardData.rarity (int) — 后端已定义，前端读取显示
   - CardMergeSystem.Merge(cardA, cardB) — 等后端实现，前端先Mock

### 验收标准
- [ ] 卡牌边框/名字按稀有度变色
- [ ] 合成面板可打开，拖拽交互正常
- [ ] 合成动画流畅（DOTween）
- [ ] 不足2张时按钮禁用

## FE-22 装备套装激活状态UI [P1] [预估 ~120行]

### 背景
EquipPanel已有套装信息展示，但套装激活逻辑后端未实现。前端先做激活状态的视觉反馈。

### 任务

   - 在套装名称旁显示进度 "2/4"
   - 数字颜色: 未达标灰色，达标绿色，超额金色

2. **套装激活动画**
   - 激活时: 套装边框金色发光脉冲（1次，0.5s）
   - 未激活: 套装区域半透明 + 灰色遮罩

3. **装备详情联动**
   - 点击已装备的套装件，高亮同套装的其他槽位
   - Tooltip 显示当前激活/待激活效果

### 接口依赖

### 验收标准

## FE-24 新手引导UI交互层 [P1] [预估 ~300行]

### 背景
TutorialSystem已有逻辑层(429行)，定义了引导步骤/触发条件/进度。但缺UI层——遮罩、高亮、手指动画。

### 任务
1. **TutorialGuideOverlay 引导遮罩组件**
   - 全屏半透明黑色遮罩（alpha=0.7）
   - 目标区域挖洞（圆形/矩形可配置）
   - 挖洞边缘柔和羽化

2. **引导手指动画**
   - 手指图标从屏幕外飞入 → 指向目标 → 点击动画
   - 支持模式: 点击/拖拽/滑动/等待

3. **引导气泡对话**
   - 在目标区域附近弹出对话气泡
   - 气泡位置自适应（上/下/左/右，避开遮挡）
   - 打字机效果逐字显示

4. **与 TutorialSystem 对接**
   - 监听 TutorialSystem.OnStepStarted / OnStepCompleted
   - 根据步骤配置自动定位目标UI元素
   - 步骤完成后自动切换到下一步

### 接口
```csharp
// TutorialSystem 已有，前端调用:
TutorialSystem.Instance.GetCurrentStep(); // → TutorialStep
TutorialSystem.Instance.CompleteStep();   // 完成当前步骤
TutorialSystem.Instance.SkipAll();        // 跳过全部引导
```

### 验收标准
- [ ] 遮罩挖洞正确指向目标UI
- [ ] 手指动画流畅
- [ ] 对话气泡不遮挡目标
- [ ] 步骤自动流转
