## 1. 构建目标

**最终目标**：微信小游戏（基于WebGL）
**当前阶段**：WebGL基线构建 → 先跑通浏览器版，再适配微信小游戏容器

## 2. 构建配置

### 2.1 Unity Player Settings（WebGL平台）

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **分辨率** | 720×1280（竖屏） | 微信小游戏标准竖屏 |
| **全屏模式** | Fullscreen Window | 自动适配屏幕 |
| **Color Space** | Linear（当前已设） | WebGL 2.0支持 |
| **Graphics API** | WebGL 2.0 only | 微信小游戏强制WebGL 2.0 |
| **Scripting Backend** | IL2CPP | 微信小游戏必须（不支持Mono） |
| **API Compatibility** | .NET Standard 2.1 | 体积最小 |
| **Managed Stripping Level** | High | 减少包体 |
| **Code Stripping** | Enabled | 微信小游戏对包体敏感 |

### 2.2 WebGL Template

使用 **Minimal** 模板（不选Default，减少不必要的HTML/CSS）。

如果微信小游戏SDK有专用模板，后续切换。当前先用Minimal验证构建。

| 参数 | 值 |
|------|-----|

## 6. 微信小游戏适配预留

### 6.1 微信SDK接入（后续阶段）

```csharp
// 后续需要接入的微信API

// - wx.shareAppMessage() → 分享
```

### 6.2 微信小游戏特定限制

- **不支持** `System.Net.WebRequest`（需改用微信网络API）
- **不支持** `System.IO.File`（需改用微信存储API）
- **不支持** `PlayerPrefs`（需适配微信本地存储）
- **不支持** 动态加载DLL
- **首包**必须 < 20MB（含WASM + Data + 首场景资源）

## 8. 可能遇到的问题

| 问题 | 解决方案 |
|------|----------|
| IL2CPP构建慢 | 正常，首次需10-20分钟 |
| Brotli压缩需Python | 确保环境有Python，或改用Gzip |
| DOTween WebGL兼容 | DOTween已支持WebGL，无额外处理 |
| Newtonsoft.Json在IL2CPP下报错 | 需加 link.xml 排除剥离 |
| Resources.Load性能 | WebGL下Resources加载正常，但避免大量资源 |

### 必须的 link.xml（防IL2CPP剥离）

```xml
<linker>
  <assembly fullname="Assembly-CSharp" preserve="all"/>
  <assembly fullname="Newtonsoft.Json" preserve="all"/>
  <assembly fullname="DOTween" preserve="all"/>
</linker>
```

放置位置：`Assets/link.xml`
