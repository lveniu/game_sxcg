# UI/UX 设计文档

> 界面风格：简洁明亮的轻量级Roguelike风格
> 核心原则：一屏一主题，减少玩家认知负担

---

## 1. 界面流程图

```
[主菜单] → [英雄选择] → [战斗准备] → [战斗观战] → [结算/商店/事件] → 循环
                ↓
         [游戏结束] ← 失败
```

---

## 4. 核心操作流程

### 4.1 召唤英雄
```
1. 点击手牌中的英雄卡
2. 系统检查人口上限
3. 检查骰子点数是否足够
4. 消耗点数
5. 在棋盘上生成英雄
6. 标记卡牌为已使用
```

### 4.2 打出卡牌
```
1. 拖拽卡牌到打出区域
2. 检查骰子点数
3. 应用卡牌效果
4. 消耗点数
5. 移除卡牌或标记已使用
```

### 4.3 卡牌合成
```
1. 长按第一张卡牌
2. 拖拽到第二张同名卡牌上
3. 系统检查合成条件
4. 合成后生成更高级卡牌
5. 播放合成特效
```

## 6. 动画与反馈

### 6.1 必要动画
| 动画 | 时长 | 说明 |
|------|------|------|
| 卡牌合成 | 0.5s | 两张卡合并成一张，放光特效 |
| 英雄召唤 | 0.3s | 卡牌变成英雄落地 |
| 攻击命中 | 0.2s | 受击方闪白+振动 |
| 伤害数字 | 1.0s | 从受击点飘起消失 |
| 战斗胜利 | 1.0s | 全屏光效+胜利文字 |
| 游戏失败 | 1.0s | 全屏暗化+失败文字 |
| 骰子滚动 | 0.5s | 骰子旋转停止 |
| 进化变身 | 1.5s | 英雄光芒大放后变形 |

### 6.2 反馈机制
- 操作成功: 轻微振动 + 音效 (占位)
- 操作失败: 红色闪烁提示
- 重要事件: 全屏光效
