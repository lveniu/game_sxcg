# 微信小游戏上架资质与流程清单

> **版本**: v1.0  
> **日期**: 2026-05-09  
> **状态**: 待审核确认

---

## 一、资质要求

### 1.1 主体要求
| 资质类型 | 要求 | 说明 |
|---------|------|------|
| 企业主体 | 营业执照 + 企业对公账户 | 小游戏主体必须是企业（个体工商户/个人不行） |
| 游戏版号 | ✅ 必填（上架时必须） | 国内上架小游戏必须有版号，无版号无法通过审核 |
| ICP备案 | ✅ 必填 | 游戏内若含网页功能需ICP备案 |
| 软件著作权 | ✅ 必填 | 申请版号的必要条件 |

### 1.2 账号注册
| 项目 | 链接 | 费用 |
|------|------|------|
| 微信公众平台 | mp.weixin.qq.com | 300元/年（认证费） |
| 微信小游戏 | 公众号后台→小游戏入口 | 免费（注册小游戏服务号时自动开通） |

### 1.3 必需材料清单
| 材料 | 用途 | 备注 |
|------|------|------|
| 营业执照照片 | 注册企业公众号 | 彩色扫描件 |
| 法人身份证正反面 | 注册企业公众号 | 彩色扫描件 |
| 对公账户信息 | 300元认证费 | 对公转账 |
| 游戏名称（≥2个） | 小游戏注册 | 避免重名，准备多个备选 |
| 游戏截图（5张） | 审核材料 | 1080×1920竖屏 |
| 游戏描述 | 审核材料 | 100字以内 |
| 隐私政策链接 | 审核材料 | 网页形式，必须包含数据处理说明 |
| 游戏版号 | 上架审核 | **最大前置条件** |
| 软件著作权证 | 申请版号 | 提前准备 |

## 二、上架流程

```
注册企业公众号(1天)
    ↓
注册小游戏服务(半天)
    ↓
申请版号(3-6个月) ← 最长前置
    ├─ 软著申请(2-3个月)
    ├─ 版号申请(1-3个月)
    └─ 材料准备：游戏截图/描述/测试包
    ↓
上传游戏版本(1天)
    ├─ WebGL构建产物
    ├─ 分包资源上传CDN
    ├─ 小游戏中后台配置
    └─ 隐私政策配置
    ↓
提交审核(1-7个工作日)
    ├─ 自动审核（通常1-3天）
    └─ 人工审核（可能加审到7天）
    ↓
审核通过 → 上线
    ↓
版本更新循环(每次迭代)
```

### 2.1 技术上传流程
| 步骤 | 操作 | 平台 | 时长 |
|------|------|------|------|
| 1 | 微信开发者工具导入项目 | 本地 | 10分钟 |
| 2 | 配置小游戏AppID | 微信开发者工具 | 5分钟 |
| 3 | 小游戏适配层调试 | 开发者工具 | 1-2小时 |
| 4 | 小游戏真机预览 | 真机 | 30分钟 |
| 5 | 上传代码包 | 开发者工具上传 | 10分钟 |
| 6 | CDN分包资源上传 | 腾讯云COS/阿里云OSS | 30分钟 |
| 7 | 小游戏中后台配置 | 公众平台 | 30分钟 |
| 8 | 提交审核 | 小游戏中后台 | 10分钟 |

### 2.2 审核注意事项
| 项目 | 要求 | 常见拒审原因 |
|------|------|-------------|
| 包体大小 | 首包≤4MB（含基础代码） | 首包超限 |
| 分包大小 | 总包≤200MB | 超出上限 |
| 隐私政策 | 必须包含，需用户同意 | 缺失或不符合规范 |
| 付费引导 | 不可诱导分享/强制付费 | 违规诱导 |
| 内容合规 | 无不良内容 | 涉及暴力/色情等 |
| 游戏功能 | 必须能正常运行 | 崩溃/黑屏/无响应 |
| 社交分享 | 分享卡片需合规 | 分享链接违规 |

## 三、版号申请（最长前置环节）

### 3.1 版号申请路径
```
软件著作权申请 → 出版单位审核 → 提交版号申请 → 审批 → 下发版号
    (2-3个月)        (1-2周)            (1-3个月)          (即时)
```

### 3.2 版号申请所需材料
- 游戏软件源代码（前后各30页，共60页）
- 游戏截图（≥10张，含所有界面）
- 游戏剧情说明
- 游戏功能说明
- 软件著作权证书
- 出版单位盖章的申请函
- 企业法人营业执照

### 3.3 版号替代方案（如短期内无法取得）
| 方案 | 说明 | 风险 |
|------|------|------|
| 海外上架 | 上架海外微信小游戏（无需版号） | 无法覆盖国内用户 |
| H5页面 | 通过H5入口间接玩 | 无社交裂变、体验降级 |
| 内测阶段 | 以测试包名义上线，不公开推广 | 用户量受限 |

## 四、TapTap 平台

### 4.1 开发者入驻
| 项目 | 说明 |
|------|------|
| 入口 | tap.cn → 开发者中心 |
| 主体 | 企业或个人均可（企业账号权限更高） |
| 认证 | 企业认证需营业执照（免费） |
| 应用发布 | 入驻后可发布游戏 |

### 4.2 上架流程
```
开发者入驻(半天)
    ↓
创建游戏条目(半天)
    ↓
提交审核(资料+截图)
    ↓
审核通过(1-3天)
    ↓
发布页面 → 积累预约/收藏
    ↓
上线后更新版本
```

### 4.3 TapTap 注意事项
- TapTap 不审核游戏版号，只审核内容合规
- 可以提前发布"预约页面"，积累用户
- 社区功能强，适合开发日志和玩家互动
- 不需要上传完整包体，可先发布概念页面
