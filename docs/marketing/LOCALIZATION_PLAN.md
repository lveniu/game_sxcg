## 二、文本提取流程

### 2.1 技术实现方案

**方案A（推荐）：ScriptableObject + JSON**
```csharp
// 创建 TextAsset 资源管理器
public class LocalizationManager : MonoBehaviour {
    public Dictionary<string, string> localizationTable;
    
    public string GetText(string key) {
        return localizationTable.ContainsKey(key) ? localizationTable[key] : key;
    }
}
```

**方案B：CSV/Excel + 导入工具**
- 适合大量文本编辑
- 需要开发导入工具
- 适合团队协作

**方案C：Unity Addressables + JSON**
- 利用现有Addressables系统
- 支持动态加载
- 无需额外工具

## 三、文化审查要点

### 3.1 游戏内文化风险

| 风险类型 | 注意 | 处理方式 |
|----------|------|---------|
| 赌博暗示 | "骰子"本身可能被视为赌博元素 | UI强调"策略"而非"赌博" |
| 宗教/文化敏感 | 避免特定宗教符号或敏感历史 | 设计阶段审查 |
| 暴力内容 | 战斗描述避免过于血腥 | 简化战斗文字描述 |
| 数字/颜色 | 部分地区对数字4有忌讳 | 国际化处理 |

### 3.2 文案调整建议

| 原文案 | 调整后 | 理由 |
|--------|--------|------|
| 骰子赌狗快感 | Dice Strategy Combo | 避免赌博暗示 |
| 差一点，就差一点！ | So Close, So Close! | 保留原意但更国际化 |
| 三条/顺子 | Three-of-a-Kind / Straight | 保留扑克术语专业性 |
| 遗物 | Relics | 游戏术语通用 |
